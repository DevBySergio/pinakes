# Pinake Manual Storage Contract

Use this reference when Pinake VS Code commands are unavailable and direct file edits are required.

## Required Layout

```text
.pinake/
  .gitignore
  pinake.json
  docs/
  .state/
    modules.json
    ui.json
    indexes.json
    migrations.json
    version.json
```

`.pinake/tools/validate-pinake.mjs` and `.github/workflows/pinake-validate.yml` are optional CI artifacts.

## Manifest Shape

Create `.pinake/pinake.json` with:

- `version`: use `1`.
- `storage.root`: use `.pinake/docs`.
- `storage.hiddenFromExplorer`: `true` only if `.vscode/settings.json` excludes `**/.pinake`.
- `project.name`: human project name.
- `project.documentationType`: usually `internal`.
- `project.audience`: string array such as `["developers", "operators"]`.
- `project.template`: setup template id.
- `modules`: object of enabled core and generated modules.
- `documents`: manifest-backed documents under `.pinake/docs`.

Each document record requires `id`, `title`, `path`, `type`, `status`, and `order`. Paths are relative to `.pinake/docs` and must not start with `/` or contain `..`.

## Markdown Frontmatter

Every Markdown document should start with frontmatter aligned with the manifest:

```markdown
---
title: "Document Title"
type: reference
status: draft
order: 1
---

# Document Title
```

Valid types are `overview`, `tutorial`, `how-to`, `reference`, `explanation`, `architecture`, `adr`, `runbook`, `changelog`, `roadmap`, `glossary`, `troubleshooting`, `testing`, and `process`.

Valid statuses are `draft`, `in-review`, `stable`, and `deprecated`.

## Direct Edit Procedure

1. Search `.pinake/docs` and `.pinake/pinake.json` before adding new documents.
2. Create or edit Markdown under `.pinake/docs`.
3. Add or update the matching manifest entry.
4. Add frontmatter aligned with the manifest.
5. Run `Pinake: Repair` when available, or run the generated validator if present.
6. Run `node resources/skills/pinake/scripts/pinake-docs-helper.mjs normalize-frontmatter --root <workspace> --write` only when using the packaged skill source locally and you need deterministic frontmatter repair.

## Validation Notes

- `Pinake: Validate` checks required files, JSON schemas, manifest paths, missing manifest documents, ADR names, Markdown style, local links, frontmatter alignment, and secret hygiene.
- The standalone validator generated by `Pinake: Generate CI Validation Workflow` mirrors the same warning-only frontmatter and secret hygiene checks.
- Warnings do not fail validation; errors do.
